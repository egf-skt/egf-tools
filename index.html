eGangotri Foundation Tools for Archival Use

A focused suite for archivists and digitization teams. Streamline PDF/TIFF conversion, bulk uploads with queue monitoring, and command-line automation—backed by a Node.js backend and MongoDB. These tools help you keep large-scale archival workflows consistent, verifiable, and auditable.

Key capabilities for Archivists

PDF & TIFF Workflow

PDF Manipulation and Conversion

Production-grade TIFF→PDF pipelines with verification and post-conversion tallies.

  • Convert TIFF batches to PDFs: pnpm run convert
  • Merge PDFs for volumes/collections: Gradle merge(mega)
  • Move merged PDFs into targets: pnpm run move-merged-pdfs
  • Post-conversion verification: pnpm run tally-post-conversion (folders/PDFs)
  • Page-count cross-checks: Gradle tally(mega)
  • Archival publishing step: Gradle uploadToArchive
Bulk Operations

Bulk Upload & Queue Monitoring

Monitor and manage large ingest jobs with MongoDB-backed queues and a web-based monitor.

  • Upload Monitor Web App for real-time status
  • Queue collections and tallies (e.g., archive_upload_monitor, Items_Queued)
  • Command-line executor: npm run exec for scripted batch jobs
Naming Pipelines

Bulk Renaming Workflows

Standardize collection naming at scale. Integrate your own rules or bring AI-assisted steps into your pipeline for consistent, human-reviewable filenames.

  • Policy-driven renaming conventions
  • Optional AI-assisted suggestions (bring-your-own model/integration)
  • Dry-run and audit logs to verify changes before commit
Platform

Backend & Infrastructure

Node.js backend with MongoDB (local or Atlas) powering the suite and monitoring services.

  • Local development: MongoDB, pnpm, nodemon
  • Production: MongoDB Atlas cluster support
  • Optional Docker access for the web UI at http://localhost:80/

Getting Started (from code)

Prerequisites

  • MongoDB (local) or MongoDB Atlas
  • pnpm and Node.js toolchain
  • nodemon (global)

Quick notes

  • Install tools: pnpm install, then npm install --location=global --force nodemon
  • Local Mongo URL example: mongodb://localhost:27017/ (Compass)
  • Collections: archive_upload_monitor, Items_Queued
  • Start backend with nodemon + pnpm, start frontend at http://localhost:3000/

Access (Docker image)

Web UI: http://localhost:80/
Ensure you are logged in with the correct account (e.g., trust@gmail.com) to avoid access denied.

Troubleshooting

Port 80 permission denied on WSL

If you see Error: listen EACCES: permission denied 0.0.0.0:80, switch from WSL terminal to a standard Bash/Command prompt.

MongoDB Atlas connection issues

If you get querySrv EREFUSED, add your IP to the Atlas Access List.
